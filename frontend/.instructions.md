# Frontend Development Guidelines

This applies to all frontend code in the `frontend/` directory.

## Technology Stack
- **Framework**: React 19
- **Language**: TypeScript (strict mode)
- **Build Tool**: Vite
- **Package Manager**: Yarn
- **UI Library**: Material-UI (MUI)
- **State**: React Query (TanStack) + Redux Toolkit
- **Forms**: React Hook Form
- **Routing**: React Router v7
- **Styling**: Emotion (via MUI)

## File Organization

### `/src` Directory Structure
- `components/` - Reusable UI components
- `pages/` - Page components (route pages)
- `application/` - App-level context and setup
- `services/` - API services and utilities
- `helpers/` - Utility functions
- `styles/` - Global styles and theme
- `stories/` - Storybook stories

### Component Structure
```
ComponentName/
  ├── ComponentName.tsx
  ├── ComponentName.scss (if needed)
  ├── useComponentNameLogic.ts (custom hooks)
  └── types.ts (TypeScript types)
```

## Key Patterns

### API Integration
- Auto-generated API client from backend OpenAPI spec
- Use React Query hooks (auto-generated)
- Example: `const { data } = useGetUsers()` from generated hooks
- See `docs/Client-autogeneration.md` for regeneration

### Forms
- Use React Hook Form for all forms
- MUI components with form integration
- Example:
  ```typescript
  const { control, handleSubmit } = useForm<FormData>();
  <Controller control={control} name="field" render={({ field }) => ...} />
  ```

### State Management
- **Server State**: React Query (automatic caching, refetching)
- **Client State**: Redux Toolkit (theme, user preferences)
- **Form State**: React Hook Form (temporary form data)

### Routing
- React Router v7 with TypeScript
- Define routes in `App.tsx` or routing module
- Use `<Link>` and `useNavigate()` for navigation

### Error Handling
- Error boundaries for component-level errors
- Suspense boundaries for async components
- React Query error states accessible via `isError`, `error`

## Code Style

### TypeScript
- **No `any` types** - use proper types or `unknown` if necessary
- Define interfaces for all data structures
- Use discriminated unions for type safety
- Generic functions with proper type constraints

### Naming Conventions
- Components: PascalCase (`UserCard.tsx`)
- Files matching components: PascalCase
- Utilities/hooks: camelCase (`useUserData.ts`)
- Constants: UPPER_SNAKE_CASE
- Non-exported functions: prefixed with underscore if private (`_helperFunction`)

### React Patterns
- Functional components with hooks (no class components)
- Custom hooks for reusable logic
- Memoization with `React.memo` for heavy components
- `useCallback` for stable function references in hooks
- `useMemo` for expensive computations

## Component Examples

### Functional Component with Hooks
```typescript
interface UserCardProps {
  userId: string;
}

export const UserCard: React.FC<UserCardProps> = ({ userId }) => {
  const { data: user, isLoading } = useGetUser(userId);

  if (isLoading) return <Skeleton />;
  if (!user) return <Alert severity="error">User not found</Alert>;

  return (
    <Card>
      <CardContent>
        <Typography>{user.name}</Typography>
      </CardContent>
    </Card>
  );
};
```

### Custom Hook for Logic
```typescript
export const useAddUser = () => {
  const mutation = useCreateUser();
  const [error, setError] = React.useState<string | null>(null);

  const handleAdd = async (data: CreateUserInput) => {
    try {
      setError(null);
      await mutation.mutateAsync(data);
    } catch (err) {
      setError(err instanceof Error ? err.message : 'Unknown error');
    }
  };

  return { handleAdd, isLoading: mutation.isPending, error };
};
```

## Styling

### Using MUI with Emotion
- Leverage MUI's `sx` prop for component styling
- Use `styled()` from Emotion for custom styled components
- SCSS for global/page-specific styles

### Example
```typescript
const StyledContainer = styled(Box)(({ theme }) => ({
  padding: theme.spacing(2),
  backgroundColor: theme.palette.background.paper,
}));
```

## Testing in Frontend

- Component tests in `frontend/tests/`
- E2E tests in `e2e/tests/` with Playwright
- Mock API responses with React Query in tests
- See `docs/UI-tests.md` for test guidelines

## Performance

- Code splitting via Route-based dynamic imports
- Lazy load heavy components with `React.lazy()`
- Use React Query for request deduplication
- Monitor bundle size: see `docs/How-to-analyze-&-reduce-bundle-size.md`

## i18n Localization

- Use `useTranslation()` hook from `react-i18next`
- Translation files in public assets
- Example: `const { t } = useTranslation(); <span>{t('key')}</span>`
- See docs for adding new languages

## Debugging

- Browser DevTools React/Redux extensions
- React Query DevTools (development only)
- Check network tab for API calls
- Use `console.log` with feature flagged blocks

## Common Patterns to Avoid

- ❌ Complex nested ternary operators
- ❌ Inline arrow functions that cause re-renders  
- ❌ Mixing async/await with `.then()` chains
- ❌ Large components (>400 lines) - split into smaller pieces
- ❌ Prop drilling more than 2-3 levels deep
- ❌ Direct DOM manipulation (use React/MUI)
