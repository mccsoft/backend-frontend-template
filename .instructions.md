# Backend-Frontend Template - General Coding Guidelines

## Project Overview
This is a full-stack web application template combining:
- **Backend**: .NET 10 ASP.NET Core (C#)
- **Frontend**: React 19 with TypeScript
- **Testing**: Playwright E2E tests
- **Deployment**: Docker + Kubernetes

See [PROJECT-STRUCTURE.md](PROJECT-STRUCTURE.md) for detailed organization.

## General Principles

### Code Quality
- Prefer readability over cleverness
- Use meaningful variable and function names
- Add comments for complex logic or business rules
- Keep functions small and focused (single responsibility)

### File Organization
- Group related code by feature/domain
- Keep utilities in appropriate `helpers/` or `services/` directories
- Colocate components with their styles when appropriate
- Avoid deeply nested directory structures (max 3-4 levels)

### Dependencies
- Frontend: See `frontend/package.json` for approved libraries
- Backend: See `webapi/Directory.Packages.props` for centralized package versions
- Do NOT add new dependencies without checking existing solutions first

### Type Safety
- **Frontend**: Use TypeScript strictly - no `any` types without justification
- **Backend**: Use nullable reference types (`#nullable enable`)
- Both: Ensure type definitions are complete before implementation

### Testing
- Write tests for critical business logic
- E2E tests in `e2e/tests/` for user workflows
- See `docs/UI-tests.md` and `docs/Auto-tests.md` for testing guidance

## Debugging & Development

### Logging
- **Backend**: Use Serilog (configured in startup)
- **Frontend**: Use `js-logger` library
- Set appropriate log levels (INFO in production, DEBUG in development)

### Error Handling
- **Frontend**: Use error boundaries in React components
- **Backend**: Return appropriate HTTP status codes with error details
- Always handle rejections/exceptions gracefully

## Common Tasks

### Running Locally
```bash
yarn install          # Install dependencies
yarn start           # Start dev environment (backend + frontend)
```

### Building for Production
```bash
yarn build          # Build Docker image
```

### Making Authentication Changes
See `docs/Authentication.md` for OpenIddict configuration and OAuth setup.

### Database Migrations
See `webapi/` README for EF Core migration commands.

### Adding API Endpoints
1. Create endpoint in backend
2. Expose OpenAPI schema (auto-documented)
3. Regenerate frontend client: see `docs/Client-autogeneration.md`
4. Use generated React Query hooks in frontend

### Modifying Localization
See `docs/Logging.md` and i18next configuration for message updates.
